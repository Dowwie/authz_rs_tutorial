authz.rs Global Statics

Global Statics vs Constants

static PART_DELIMETER: &'static str = ":";
static SUBPART_DELIMETER: &'static str = ",";

A string primitive is represented in Rust as &'static str. Declaring that string as static makes the string accessible throughout the authz module and does not inline the variable upon use (it gets a fixed memory location). Statics live for the entire lifetime of the Rust program. Note the use of the single apostrophe following the ampersand: this denotes the use of a lifetime.

This project chose static rather than const for no particular reason. There was no requirement here for a memory location associated with a constant. Rather than static, the Rust book recommends const:

const PART_DELIMETER: &'static str = ":";
const SUBPART_DELIMETER: &'static str = ",";
